### LAX Intraday Floor Model (near_forecast_signal)

**Goal:** Predictive floor signal for LAX (KLAX) temperature markets. During 8–11 AM LST while temp is rising and OM still has upside, buy YES/NO on thresholds 1-3°C above current reading before bots react to the physical crossing.

**v1/v2 model: scrapped.** Two sub-agents ran (lax-trajectory-model, lax-next-hour-model). Built probability tables with P(δ1h ≥ 1°F) ≈ 95.7% at 9 AM under good slope/OM conditions. But: trained on integer °F from IEM asos1min, which does NOT match live wethr data (whole °C). Different quantization levels → training/production mismatch.

**The core unit system problem:**
- IEM asos1min returns integer °F (native US ASOS)  
- Live wethr returns whole °C
- Training slope in integer °F = 1°F steps; production slope in °C = 1.8°F steps
- These are not compatible — the model was calibrated on data that doesn't look like live data

**v3 model (spawned as lax-model-v3):** Train entirely in °C. 
- Convert 1-min integer °F → whole °C: `round((tmpf-32)*5/9)` to simulate 5-min wethr
- Predict `next_hour_max_C` in whole °C  
- Additional features: hourly METAR decimal °F (0.1°C precision as anchor), rolling 6-hr high in °C
- At trade time: map predicted max_C back to °F ranges using standard ASOS °C→°F disambiguation
- Re-fetch: IEM hourly ASOS for KLAX (decimal °F via `asos.py`, not asos1min)
- Sub-agent label: `lax-model-v3` — will auto-announce when done

**near_forecast_signal disabled until v3 completes:**
- `_NEAR_FORECAST_CITIES = frozenset()` in nws_obs_trader.py
- The signal code/structure stays but fires for no cities

**Important insight from Ian:** The 6-hour high/low from wethr push events has higher precision than 5-min whole °C. During rising temps, six_hr_high ≈ current temp but more precise. Also: hourly METAR (0.1°C) can serve as a precision anchor beyond the 5-min quantized reading.
