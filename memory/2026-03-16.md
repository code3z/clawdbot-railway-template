## ⚠️ Return Calculations — ALWAYS USE COMPOUND, NEVER SIMPLE (2026-03-16)

**Ian corrected me 4 times in one conversation. Do not repeat this.**

**WRONG:** `r% × 365` = "annualized return"  
**RIGHT:** `(1 + r)^365 - 1` = annualized compounded return (CAGR)

Examples:
- 1% daily → `(1.01)^365 - 1 = 37.8x = 3,678%` annualized (Ian's example)
- 2% daily → `(1.02)^365 - 1 ≈ 1,377x` annualized
- Never say "X% daily = Y% annualized" using simple multiplication

For short-duration trades, just state the per-trade return directly (e.g. "2¢ profit on 98¢ in one day") rather than annualizing, which is misleading anyway since the same capital can't be redeployed 365 times per year in the same trade.

## twc_portfolio_trader — built and running (Mon Mar 16)

### What it is
- Paper trader alongside twc_morning_trader
- Model name: `twc_portfolio`
- File: `polymarket/twc_portfolio_trader.py`
- Added to `ALL_MODELS` in `paper_trading/db.py` and `SIM_MODELS` in `dashboard_data.py`

### Design
- **Continuous loop**: every 60s, fetches fresh live prices, recomputes edge for all 20 cities
- **TWC refresh**: every 30 min (markets don't change, only prices)
- **Market list**: cached 3h (markets are static for tomorrow)
- **Portfolio-level allocation**: build optimal desired set top-down by edge, total deployable cash = PAPER_BANKROLL - $2 reserve
- **8% market cap**: per market = city/date/measure combined (filled + resting), never filled positions are cancelled
- **Initial placement**: ask - offset (5¢ early, 3¢ mid, 2¢ late, 0¢ near deadline) — NEVER at max_limit
- **Repricing**: phase-based same as twc_morning: bidb <5pm, bida 5-9pm, take_it 9-10pm
- **Step-up guard**: 30+ depth needed for thin edge, 2+ for strong (>20%) edge
- **Cooldown**: 60s between reprices on same ticker
- **Hysteresis**: only displace resting order if replacement has >2% better edge
- **Deadline**: 10 PM ET

### Key bugs found and fixed
1. v1: `_place_paper_order` hardcoded `limit_price = _cp(our_p - MIN_EDGE)` = max_limit — caused LV to thrash between 49¢ and 75¢ every loop. Fixed by passing explicit limit_price.
2. v1: `_find_cancels` only cancelled on full edge loss, not displacement — fixed with full portfolio rebalance + hysteresis.
3. v1: No step-up/down logic at all — added phase-based repricing matching twc_morning.
