### °C disambiguation research completed tonight

**Empirical base rates** (60 ambiguous city-dates, Feb 22-26, 12 cities):
- HI (upper °F): 55% — NOT 57% as initially reported (formula was wrong in analysis scripts)
- LO (lower °F): 36%
- MISS (1-min peak exceeded 5-min range): 9%
- Correct conversion function: `c_to_possible_f()` in nws_obs_trader.py — verified correct
- Old analysis scripts used `floor(...+0.5)` which was wrong for some values

**METAR anchor works only when ≤10 min from peak:**
- ≤10 min: 78% correct (n=18)
- 10-20 min: 0% correct (n=4)
- 20-35 min: 40% correct (n=5)
- 35+ min: 17% correct (n=6)

**NWS API has 5-min historical data** (~7 day rolling window only)
**asos1min** (Iowa State) has 1-min °F data going back years (spotty coverage)
**IEM ASOS** only has hourly METARs historically (5-min slots are M/missing)

**Reconstruction algorithm** (verified correct against NWS live data):
- 5-min obs at time T uses 1-min readings T-4 through T (window ENDS at T)
- avg 5 readings → round to whole °F → convert: round((F-32)*5/9) = whole °C
- Verified: KMDW 18:00 = -4°C from 17:55-17:59 window [25,25,25,25,25]°F
