### Key Findings
1. **`noon_price` dominates both models**: coef=5.37 in NO model — far larger than all other features. Model is essentially "trust the market + small TWC correction." Could mean market is well-calibrated, or training set too small to learn city effects.
2. **Top features NO**: noon_price (+5.37), log_delta (+0.57), strike_between (-0.48)
3. **Top features YES**: noon_price (+1.45), strike_greater (+0.64), strike_less (-0.54)
4. **City bias stats (training set, high markets)**:
   - Cold-biased (TWC over-predicts high): Denver -1.28°F, Las Vegas -1.33°F → explains good NO/less performance
   - Warm-biased (TWC under-predicts high): SFO +1.89°F, Boston +1.72°F, NYC +1.50°F, Miami +1.50°F → explains losses
5. **Calibration is rough**: NO model lowest bin: predicted 0.57 → actual 0.00. Test window is only 4 dates (Mar 17–20) — too small to trust. Need 2-3 more months.

### Winner's Curse / Selection Bias Test (ran today)
- Tested if prior=NO advantage is a selection effect (prior=NO windows opening at better mids)
- Test 1: avg mid for prior=NO = 0.6212 vs prior=YES = 0.6209, p=0.76 → no mid-price selection
- Test 3: BTC-vs-ref +0.0008% vs -0.0080%, p=0.73 → no positional selection
- Test 2 (key): at same narrow mid sub-band, prior=NO beats prior=YES at all 5 sub-bands consistently (+2.6% to +19.1%), 0.62–0.63 band significant (p=0.013)
- Conclusion: selection bias not mechanically explaining edge; prior=NO appears genuinely informative
