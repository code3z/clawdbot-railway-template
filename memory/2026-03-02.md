## 6hr METAR Window Bug — Found, Fixed, Voided

### Root Cause
The 05:51Z synoptic METAR contains a `10xxx` 6-hour max group whose window spans
23:51Z yesterday → 05:51Z today. For ET stations (UTC-5 LST), 05:51Z is already
"today" locally, but the 6hr window start (23:51Z) is still "yesterday" ET.
The original date check only verified the obs timestamp was today — not that the
full 6-hour window was within today's local calendar day.

Result: at 3:17 AM ET on March 2, KNYC's 05:51Z obs showed 6hr max = 3.9°C = 39°F
(March 1 afternoon warmth), which triggered a false "confirmed high > 35°F" signal
even though KNYC never exceeded 0.6°C (33°F) on March 2.

### Fix Applied
Both `nws_obs_trader.py` and `cli_strategy/trader.py` now require:
```python
window_start_utc = obs_time_utc - timedelta(hours=6)
# Only apply if window_start is >= LST midnight today
if window_start_utc >= lst_midnight(local_date, tz_name):
    # apply 6hr augmentation
```
The first safe synoptic METAR for ET is 11:51Z (6:51 AM ET), whose 6h window
starts at 05:51Z = LST midnight ET. For CT/MT/PT this bug never manifested
because those timezones' midnight is after 05:51Z.

## Obs-vs-Forecast Deviation Model — New Trading Signal Idea

Ian suggested: trade based on how current obs compare to what the hourly forecast
predicted for right now. If obs is running +7°F warmer than OM's hourly forecast
at 10 AM, the daily high will likely also run ~7°F warm. Market prices off overnight
daily forecast — we'd know by 10 AM the day is tracking differently.

**Model:**
```
actual_high - nbm_high = β(t) × [obs(t) - om_hourly_forecast(t)]
```
β(t) varies by time of day — morning anomaly more predictive than afternoon.

**Why better than truncated normal alone:**
- Fires earlier (9-10 AM) before running max reaches tradeable territory
- Uses trajectory info, not just lower bound
- Combined: deviation model fires early, truncated normal confirms near peak
- Both complementary

**Data needed:**
- Open-Meteo historical forecast API: what model predicted for each hour on past dates
- Open-Meteo archive: actual hourly obs (already used in backtest)
- Fit β(t) per city per hour via OLS on CLI actuals
